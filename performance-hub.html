// 03 — Performance Hub

Gamificação
auditável.

FastAPI MySQL React Power BI Supabase
Contexto

Operação com 100+ consultores precisava de um sistema para apurar e exibir desempenho em tempo real, gamificando metas individuais e de equipe. O processo legado envolvia 30 minutos de fechamento manual por ciclo, planilhas Excel transitando por email e zero rastreabilidade.

A solução

Backend em FastAPI + MySQL com ~80 endpoints, frontend em React e camada analítica em Web. A regra-chave: todo evento que afeta score é imutável — fica gravado, com hash, autor e timestamp. Correções viram novos eventos. 

# FastAPI — evento de pontuacao imutavel
class ScoreEvent(BaseModel):
    consultor_id: int
    delta: int
    reason: ScoreReason
    actor_id: int
    metadata: dict = {}

@router.post("/scores/events")
def post_event(ev: ScoreEvent, db: Session = Depends(get_db)):
    record = build_record(ev, db)
    record.signature = sha256_payload(record)   # assinatura
    db.add(record)
    db.commit()
    broadcast(record)                            # supabase realtime
    return record
Destaques técnicos

Realtime sem dor

Cada evento publicado dispara um broadcast via Supabase Realtime. O front aplica o delta sem refazer requisições — a UI atualiza em ~200ms do fato à tela.

Auditoria como contrato

  • Eventos imutáveis com assinatura SHA-256 do payload.
  • Reversões são novos eventos, nunca DELETE.
  • Auditor consegue reconstruir qualquer ranking histórico.
  • Fechamento de ciclo é uma view, não uma rotina.

Power BI como camada de leitura

Os dashboards de supervisão consomem uma view materializada do MySQL via gateway. O BI nunca escreve, nunca calcula score — só lê.
