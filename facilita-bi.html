// 01 — Fac. BI

Ecossistema
Analítico.

Power BI DAX Power Query (M) Excel
Contexto

Uma das maiores operações de telemarketing 0800 do Brasil precisava entregar análise para três audiências diferentes usando o mesmo modelo de dados: diretoria da operação, supervisores de equipe e o cliente final (a associação contratante). Cada audiência via números diferentes — e os dados de um cliente nunca podiam vazar para o outro.

A solução existente era um mosaico de planilhas, com fechamentos mensais artesanais. Lento, frágil e impossível de auditar.

A solução

Um BI multi-tenant com isolamento de dados na camada de Power Query: parâmetro de tenant injetado no RLS, filtros propagados na origem, e três personas de relatório consumindo o mesmo modelo semântico. 

// Power Query — filtro por tenant na origem
let
    Source = Sql.Database("warehouse", "facilita"),
    Tenant = Parameter.Tenant,
    Filtered = Table.SelectRows(
        Source{[Schema="dw", Item="fato_atendimento"]}[Data],
        each [tenant_id] = Tenant
    )
in
    Filtered

Três app workspaces distintos no Power BI Service, cada um com sua RLS rule, alimentados pelo mesmo dataset certificado. Atualização incremental semanal para os dashboards de operação.

Destaques técnicos

Métricas com tempo dinâmico

DAX com SELECTEDVALUE e SWITCH para que o mesmo card calcule MTD, QTD ou YTD conforme o filtro de calendário do usuário — sem duplicar medidas no modelo. 

-- DAX — receita time-aware
Receita Periodo =
VAR Periodo = SELECTEDVALUE(d_Periodo[Codigo], "MTD")
VAR Base    = [Receita Bruta]
RETURN
    SWITCH(
        Periodo,
        "MTD", CALCULATE(Base, DATESMTD(d_Calendario[Data])),
        "QTD", CALCULATE(Base, DATESQTD(d_Calendario[Data])),
        "YTD", CALCULATE(Base, DATESYTD(d_Calendario[Data])),
        Base
    )

Decisões de arquitetura

  • Star schema enxuto: 1 fato, 6 dimensões. Sem snowflake.
  • Camadas de pasta no Power Query: 0_Source → 1_Stage → 2_Clean → 3_Model.
  • Convenção de nomes consistente: f_* para fatos, d_* para dimensões, _med_* para grupo de medidas.
  • Documentação inline em cada query, em português, com decisão e contexto.

O fechamento mensal, antes feito em planilhas e revisado durante dias, virou um relatório consultado em tempo real pela diretoria. Os supervisores deixaram de pedir extrações ao time de dados — passaram a navegar o próprio recorte.

Mais importante: o modelo virou referência interna. Outros BIs da casa adotaram a mesma convenção de pastas, camadas e nomenclatura.
